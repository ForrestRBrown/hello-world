Introduction to the Web:

The internet, or web, is a three (3) part system of communicating computers. The major pieces of the web are:
  1. You, your computer, and your browser.
  2. The internet, and requests made via the internet in a protocol called "Hypertext Transfer Protocol Secure" (HTTP) that unites the Web.
  3. Servers:computers designed for storage and communication rather than personal use. Servers respond to your requests via HTTP with files, including HTML, picture, video, music, and more. HTML documents form the majority of the web, and HTTP is what unites the web.

HTML Basics:

HTML documents form the heart of the web. They consist of:

  1. Text content: what you see.
  2. Markup: styling, or what it looks like arranged on the page
  3. References to other documents, such as images and videos.
  4. "Hyperlinks" or Links.

Computers are Stupid:

Computers are stupid in the sense that they take things very literally. In this sense both computer languages and coders must take into account this "stupidity" of the computer. Errors can arise from coding that the computer cannot read, or misinterprets.

Inline vs. Block Elements:

There are multiple categories of elements in HTML. Those described in the lesson are inline, and block.
-Inline elements are those found in the code line, such as "b", "em", "img", or "span" tags.
-Block elements actually define a section within the code. This section may have other attributes like width or height. Some examples are a "p", or "div" tag.

Using Developer's Tools:

Developer's tools can be found in most browsers and are used to view the underlying code for a website.
Examining a website like Wikipedia's HTML, it can be seen that the page is divided into boxes representing different sections of the HTML document. This ties into lesson one, where we learned about the difference between "inline" and "block" elements. Visual notes and any text displayed on the page can be found with the developer's tools, although sometimes text is buried deep in <div> tags!

Decoding Boxes:

Every webpage can be broken down into a multitude of boxes depending on the complexity of the page. Programmers use indentation to indicate just how deep into any given box, or set of boxes you're reading into . For example, the whole page could be considered a box, but within the larger box there are many other boxes representing images, videos, text and more. Everything visible on a webpage is a box.

Document Object Model:

Breaking down the boxes/blocks of HTML into a diagram creates a "treelike" structure that represents the organizational structure of the document. Visually, the wave-like form of the indentations programmers use represents this structure.
This structure is referred to as the Document Object Model or "DOM"

CSS & Styling:

The lesson also begins to talk about the difference between structure and styling, or the difference between the functions of HTML and CSS. While HTML provides the structure for a document, CSS provides the styling: colors, shapes, borders, etc.
